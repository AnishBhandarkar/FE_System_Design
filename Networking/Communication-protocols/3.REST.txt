1) URL:

URL stands for  Uniform Resource Locator .
It is the address you type in a browser to access a webpage or resource on the internet.

Example:
https://www.example.com:443/products?id=10#reviews
 

---

1Ô∏è‚É£  Scheme / Protocol 
https
 
This tells the browser  how to communicate with the server.

Common schemes:

*  http  ‚Üí unsecured website
*  https  ‚Üí secured website
*  ftp  ‚Üí file transfer
*  mailto  ‚Üí open email

---

2Ô∏è‚É£  Domain (Hostname) 

www.example.com

This is the address of the server where the website is stored.

It can include:

* Subdomain ( www )
* Second-level domain ( example )
* Top-level domain ( .com )

Example:

blog.store.github.io
 

---

3Ô∏è‚É£  Port (Optional) 

:443

Port is rarely typed manually. It tells which door on the server to connect to.

Common ports:

*  80  ‚Üí HTTP
*  443  ‚Üí HTTPS
*  21  ‚Üí FTP

If not mentioned, browser uses default.

---

4Ô∏è‚É£  Path 

/products

This is the  location of a specific page or file on the server.

Examples:

*  /about-us 
*  /images/logo.png 
*  /api/user/42 

---

5Ô∏è‚É£  Query Parameters 

?id=10

Used to send  data to the server .

Format:

 
?key=value&key2=value2
 

Examples:

* Search:  ?q=mobile 
* Pagination:  ?page=2 
* Filters:  ?category=laptop&price=low 

---

6Ô∏è‚É£  Fragment (Hash) 
 
#reviews

Identifies a  specific section of the page , not sent to the server.

Examples:

* Jump to paragraph on page
* Jump to heading in documentation

---

# üìå Final Summary

| Part         | Example       | Meaning                |
| ------------ | ------------- | ---------------------- |
|  Scheme   |  https        | Communication protocol |
|  Domain   |  example.com  | Website address        |
|  Port     |  :443         | Server connection port |
|  Path     |  /products    | Page/location          |
|  Query    |  ?id=10       | Data for server        |
|  Fragment |  #reviews     | Section of page        |
